## 1. 디자인 철학

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **감정 제거** | 분노·팬덤·정치 성향을 자극하는 시각 요소 금지 |
| **검증 강화** | 모든 정보는 출처·기록·비교 가능 형태로 제공 |
| **중립적 신뢰** | 공공기관 수준의 안정감, 정당 연상 색상 배제 |
| **데이터 우선** | 사람이 아닌 행동 데이터가 UI의 주인공 |
| **조용한 강력함** | 과시형 애니메이션 없이, 정교한 마이크로 인터랙션 |

### 이 서비스가 보여야 하는 것

```
✅ 공공 데이터 플랫폼
✅ 정치 행동 검증 도구
✅ 입법 협업 시스템
✅ 신뢰 기반 정책 터미널
```

### 이 서비스가 보여선 안 되는 것

```
❌ 정치 커뮤니티 / 진영 토론장
❌ 선동형 플랫폼 / 감정 소비형 SNS
❌ 유튜브 뉴스 느낌 / 댓글 전쟁 구조
❌ 강한 정당 컬러 (빨강 vs 파랑 구도)
```

## 5. 컴포넌트 라이브러리

### RadarChart

6개 행동 지표를 육각형 레이더 차트로 시각화.

```
지표: 출석률 · 법안통과율 · 발언일치율 · 공약이행률 · 입법영향력 · 활동성
범위: 0–100 (0이 중심, 100이 외곽)
```

**Props:**

```jsx
<RadarChart
  data={[{ label: '출석률', value: 96 }, ...]}
  size={200}           // px, 기본 200
/>
```

**스타일 규칙:**

- 그리드 링: 4개 (25/50/75/100%), 100% 링은 실선, 나머지는 점선
- 데이터 영역: accent 색상 + fillOpacity 0.12
- 데이터 포인트: hover 시 툴팁 표시

---

### GaugeRing

단일 지표의 원형 게이지. 주로 종합 신뢰도 표시에 사용.

```jsx
<GaugeRing
  value={91}
  size={100}        // px
  label="신뢰도"
  color="var(--success)"
/>
```

---

### TrendLine

시계열 추세를 부드러운 곡선으로 표시. 카디널 스플라인 보간.

```jsx
<TrendLine
  data={[72, 75, 78, 82, 85, 87, 89, 91]}
  width={260}
  height={44}
  color="var(--success)"
/>
```

---

### ActivityHeatmap

GitHub 스타일 24주 활동 히트맵.

```jsx
<ActivityHeatmap
  data={[...168개 값, 0=비활동 / 0~1=활동량]}
  weeks={24}
  cell={13}
  gap={3}
/>
```

**색상 단계:**

```
0       →  var(--border)           비활동
0–0.33  →  rgba(accent, 0.28)      낮음
0.33–0.66 → rgba(accent, 0.58)    보통
0.66+   →  rgba(accent, 0.90)      높음
```

## 6. 페이지별 UX 스펙

### 6-1. 메인 랜딩 (개요)

**목적:** 플랫폼 신뢰성 전달, 주요 기능 소개

**구성 요소:**

1. 히어로 — 슬로건 + 깜빡이는 커서 (터미널 감성)
2. 플랫폼 상태 배지 — LIVE 표시 (pulse 애니메이션)
3. 통계 그리드 (4열) — 등록 의원 / 검증 법안 / 공개 소환 / 평균 신뢰도
4. 핵심 기능 카드 (3열) — 블라인드 평가 / 입법 위키 / 공개 소환
5. 최근 활동 피드 — 타임스탬프 + 활동 유형 도트

**카피라이팅 원칙:**

- "화났다" 대신 "검증했다" 유도
- 데이터·기록·검증 키워드 중심
- 감정 자극 문구 금지

---

### 6-2. 블라인드 의원 평가

**목적:** 편향 없는 순수 행동 데이터 평가

**블라인드 상태 (기본):**

- 이름 숨김 → `L-◆◆1` 형태 익명 ID
- 정당 숨김
- 사진 없음
- 행동 데이터만 표시 (레이더 차트 + 지표 바)

**공개 상태 (사용자 선택 후):**

- 이름·정당·지역구 표시
- `애니메이션: fadeIn 0.4s`로 부드럽게 전환
- "검증 완료" 배지 표시

**필터:**

- 지역권 (수도권/영남권/호남권/충청권/강원권)
- 정렬 기준 (신뢰도/출석률/발언일치율/입법영향력)
- 오름차순/내림차순 토글

**카드 상단 점수 바:**

- 카드 최상단 2px 선 = 신뢰도 비율 (0–100%)
- 색상 자동 매핑

---

### 6-3. 입법 위키

**목적:** 법안을 오픈소스 협업처럼 투명하게 추적

**레이아웃:** 2패널 (사이드바 법안 목록 + 메인 상세)

**탭 구조:**

| 탭 | 내용 |
|----|------|
| 조문 | diff 뷰 (원문 삭제선 + 수정안 녹색) |
| 수정이력 | 타임라인 (vertical timeline) |
| 기여자 | 커밋 수 + 추가/삭제 라인 수 |
| 토론 | 수정제안/근거추가/반론제시 스레드 |

**버튼 네이밍 원칙:**

```
✅ "수정 제안" / "근거 추가" / "반론 제시" / "조문 개선"
❌ "찬성" / "반대" / "좋아요" / "싫어요"
```

**Diff 색상:**

```
신설 조문:  배경 rgba(74,222,128,.07)  + 앞에 "+"  초록색
수정 조문:  배경 rgba(251,191,36,.06)  + 앞에 "~"  노란색  (원문에 취소선)
삭제 조문:  배경 rgba(248,113,113,.07) + 앞에 "−"  빨간색
```

**합의도 지표:**

- 0–100% 진행 바
- 70% 이상: var(--success)
- 70% 미만: var(--warning)

---

### 6-4. 공개 소환 시스템

**목적:** 의원에 대한 공식 공개 질의 추적

**상태 4단계:**

```
접수 ─── 발송 ─── 열람 ─── 답변
  1        2        3        4
```

| 상태 | 배지 | 색상 |
|------|------|------|
| pending | 답변 대기 중 | Warning (amber) |
| viewed | 의원실 열람 확인 | Accent (teal) |
| answered | 공식 답변 등록 | Success (green) |
| expired | 90일 미응답 | Danger (red) |

**일수 카운터 규칙:**

- 0–79일: var(--text-3) 중립색
- 80일+: var(--danger) 경고 강조

**공문 형식 원칙:**

- 질의 번호 시스템 (SU-YYYY-NNNN)
- 공식 요청 배지
- 지지자 수 표시 (민주성 근거)
- 카테고리 분류 (표결불일치/출석불이행/예산집행/이해충돌/공약불이행)

---

### 6-5. 분석 대시보드

**목적:** 개별 의원의 종합 행동 분석 리포트

**구성:**

```
┌──────────────────────────────────────┐
│ 의원 신원 + GaugeRing (신뢰도)       │
├────────────────┬─────────────────────┤
│ RadarChart     │ HorizBar (6개 지표) │
│ (6대 지표)     ├─────────────────────┤
│                │ TrendLine (8분기)   │
├────────────────┴─────────────────────┤
│ ActivityHeatmap (24주)               │
└──────────────────────────────────────┘
```

**의원 선택기:**

- 상단 우측 드롭다운
- 신원 공개 여부에 따라 이름/익명 ID 표시

## 7. 인터랙션 가이드

### 허용되는 모션

```css
/* 페이지 전환 */
animation: fadeIn .2s ease;

/* 카드 호버 */
transform: translateY(-1px);
transition: border-color .2s, transform .15s;

/* 데이터 진행 바 */
transition: width .6s cubic-bezier(.4,0,.2,1);

/* 게이지 링 */
transition: stroke-dasharray .7s cubic-bezier(.4,0,.2,1);

/* 신원 공개 */
animation: fadeIn .4s ease;
```

### 금지되는 모션

```
❌ bounce / elastic 이징
❌ 전체화면 전환 애니메이션
❌ 흔들림(shake) 효과
❌ 파티클·반짝임 효과
❌ 500ms 이상 지연 애니메이션
```

### 마이크로 인터랙션 규칙

| 요소 | Hover | Active |
|------|-------|--------|
| 버튼 | opacity .85 또는 border-color 변화 | scale .98 |
| 카드 | border-2, translateY(-1px) | — |
| 데이터 포인트 | r 증가 + 툴팁 표시 | — |
| 내비 아이템 | background: var(--border) | — |

## 8. 금지 패턴

### 시각 요소

```
❌ 정당 상징 색상을 메인 강조색으로 사용
❌ 정치인 얼굴 중심 썸네일
❌ 감정 자극형 배너 이미지
❌ 짤·밈·숏폼 영상
❌ Glassmorphism 과다 사용
❌ 강한 drop-shadow (border만 사용)
❌ 그라디언트 배경 과다
```

### 콘텐츠·카피

```
❌ "분노" "배신" "적폐" 등 감정 자극 단어
❌ 추천/비추천 전쟁 구조 (좋아요/싫어요 버튼)
❌ 선동적 제목·헤드라인 스타일
❌ 정치 진영 구도를 강화하는 프레이밍
```

### UI 패턴

```
❌ 무한 스크롤 댓글 (→ 스레드 토론으로 대체)
❌ 알림 과부하 (→ 필요한 상태 변화만 표시)
❌ 공유/확산 중심 CTA (→ 검증/기록 중심 CTA)
```
